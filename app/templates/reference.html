How It Works


The Flutter app takes an user input in the form of a drawing. This drawing is converted into base64 encoding and sent to the server.

The server at the "/predict" endpoint receives the string, decodes it and retrieves the image.

This image is then preprocessed and fed into the neural network model that generates the prediction.

(The model has been trained on the MNIST dataset.)

Deployment of The REST API


The server is a webapp built using the Flask microframework. It is easy to set up and got the work done for this project.

The Tensorflow library needs to be used for the server to make predictions. However, deployment to Heroku has certain limitations, one of them being that the slug size may not exceed 500 MB. Including Tensorflow 2.2.0 in the dependencies causes the slug size to exceed 500 MB.

It should be noted that this limitation is applicable for deployment to Heroku using Git. The alternative method used in case of deployment of this server was dockerization. A Docker image was built, which was then pushed to Heroku. There is no size limitation for Docker images.
